1) SOC là gì? Vì sao chọn Wazuh?

SOC (Security Operations Center) là "trung tâm điều hành an ninh", nơi bạn thu thập log, phát hiện bất thường, cảnh báo và điều tra sự cố.

Wazuh là nền tảng mã nguồn mở cho XDR/SIEM: dễ dựng lab, tài liệu đầy đủ, giao diện web trực quan, agent nhẹ, hỗ trợ Windows/Linux/macOS. Với lab nhỏ, bạn dùng all‑in‑one (Server + Indexer + Dashboard chung một VM) là đủ.

2) Kiến trúc lab & sơ đồ mạng

Mục tiêu: 1 máy chủ Wazuh (Ubuntu) giám sát 1 máy Windows (endpoint).

[Host laptop/PC] │ ├── (VirtualBox) │ ├── VM #1: Ubuntu Server 22.04 → Wazuh all-in-one │ └── VM #2: Windows 10/11 → Endpoint + Wazuh Agent + Sysmon │ └── Mạng: Bridged (đơn giản) hoặc NAT + Host‑only (cô lập lab)

Cổng mặc định cần dùng (xem Phụ lục): 1514/TCP (agent↔server), 1515/TCP (enrollment), 55000/TCP (Wazuh API), 443/TCP (Dashboard).

3) Chuẩn bị: phần cứng, phần mềm, file ISO

Phần cứng đề xuất (lab nhỏ: ≤5 agent)

  • CPU 4 core+
  • RAM 12–16 GB (tối thiểu có thể 8 GB nhưng sẽ chật)
  • SSD trống ≥ 60–100 GB

Phần mềm cần

  • VirtualBox (kèm Extension Pack – tuỳ chọn).
  • Ubuntu Server 22.04/24.04 ISO (64‑bit).
  • Windows 10/11 ISO (bản dùng thử/học tập tuỳ bạn).
  • Wazuh install script (trợ lý cài all‑in‑one).
  • Sysmon (bộ Sysinternals) + file cấu hình sysmonconfig phổ biến.
Mẹo: Tải trước các ISO để tiết kiệm thời gian khi dựng lab. Đảm bảo BIOS/UEFI đã bật Virtualization (Intel VT‑x/AMD‑V).

4) Tạo mạng trong VirtualBox (Bridged vs NAT + Host‑only)

Bạn có 2 lựa chọn đơn giản:

Lựa chọn A — Bridged Adapter (dễ nhất)

  • Mỗi VM nhận IP trong cùng LAN với máy thật → Laptop/PC và 2 VM "nhìn thấy" nhau trực tiếp.
  • Chỉ cần chọn Network > Adapter 1 > Attached to: Bridged Adapter cho từng VM. Chọn đúng card mạng đang dùng (Wi‑Fi/Ethernet).

Lựa chọn B — NAT + Host‑only (an toàn/cô lập hơn)

  • NAT cho Internet outbound; Host‑only để 2 VM nói chuyện với nhau và với host.
  • Bước gợi ý:
    1. VirtualBox File > Host Network Manager → Create Host‑only vboxnet0 (DHCP bật).
    2. Với mỗi VM: Adapter 1 = NAT; Adapter 2 = Host‑only (vboxnet0).
    3. Dùng IP Host‑only để Wazuh và Windows trao đổi.
Khuyến nghị: Nếu bạn viết blog cho người mới, Bridged là con đường ít rắc rối nhất. Nếu bạn muốn lab kín, chọn NAT+Host‑only.

5) Tạo VM Wazuh Server (Ubuntu)

Bước 5.1 — Tạo VM mới

  1. New → Name: wazuh-server
  2. Type: Linux, Version: Ubuntu (64‑bit)
  3. Memory: 4096–8192 MB (tuỳ RAM máy bạn)
  4. CPU: 2 vCPU (tốt hơn: 4 vCPU)
  5. Disk: VDI, Dynamically allocated, size 50–100 GB
  6. Network: chọn theo mục 4 (Bridged hoặc NAT+Host‑only)
  7. Mount ISO Ubuntu vào Storage > Controller: IDE/SATA > Choose disk

Bước 5.2 — Cài Ubuntu tối giản

  • Chọn Install Ubuntu Server
  • Ngôn ngữ/keyboard tuỳ bạn
  • Hostname: wazuh-server
  • Tạo user wazuhadmin (ghi nhớ mật khẩu)
  • OpenSSH server: Enable (để SSH tiện quản trị)
  • Partition: Use entire disk (mặc định)

Bước 5.3 — Cập nhật & tiện ích cơ bản

Đăng nhập rồi chạy:

sudo apt update && sudo apt -y upgrade
sudo apt -y install curl tar ufw jq

Bước 5.4 — (Tuỳ chọn) IP tĩnh cho VM

Nếu bạn dùng Host‑only hoặc muốn IP cố định:

ip a   # xem tên interface, ví dụ: enp0s3
# chỉnh netplan:
sudo nano /etc/netplan/01-netcfg.yaml
# ví dụ mẫu:
# network:
#   version: 2
#   ethernets:
#     enp0s3:
#       addresses: [192.168.56.10/24]
#       gateway4: 192.168.56.1
#       nameservers:
#         addresses: [1.1.1.1,8.8.8.8]

sudo netplan apply

6) Cài Wazuh all‑in‑one (Server + Indexer + Dashboard)

Cách nhanh, gọn cho lab: dùng installation assistant.

Bước 6.1 — Chạy script cài đặt

# Tải và cài đặt (all‑in‑one)
curl -sO https://packages.wazuh.com/4.14/wazuh-install.sh
sudo bash ./wazuh-install.sh -a

Kết thúc, màn hình in tóm tắt đại loại như:

INFO: You can access the web interface https://<WAZUH_DASHBOARD_IP_ADDRESS>
User: admin
Password: <ADMIN_PASSWORD>

Bước 6.2 — Truy cập Dashboard

  • Trên máy thật, mở trình duyệt: https://<IP_WAZUH>
  • Lần đầu có cảnh báo chứng chỉ tự ký → Accept/Continue.
  • Đăng nhập admin / <ADMIN_PASSWORD>
  • (Tuỳ chọn) In ra các mật khẩu thành phần:
sudo tar -O -xvf wazuh-install-files.tar \
  wazuh-install-files/wazuh-passwords.txt

Bước 6.3 — Mở cổng tường lửa (nếu bật UFW)

sudo ufw allow 443/tcp     # Dashboard
sudo ufw allow 1514/tcp    # Agent -> Server
sudo ufw allow 1515/tcp     # Enrollment
sudo ufw allow 55000/tcp    # Wazuh API
sudo ufw reload

Bước 6.4 — Kiểm tra dịch vụ

sudo systemctl status wazuh-manager
sudo systemctl status wazuh-dashboard
sudo systemctl status wazuh-indexer
# hoặc kiểm tra nhanh cổng lắng nghe
sudo ss -tulpn | egrep ':443|:1514|:1515|:55000|:9200'
Ghi chú: Muốn gỡ cài đặt, chạy lại script với -u. Với lab, bạn có thể snapshot VM sau khi cài xong để dễ quay lại trạng thái sạch.

7) Tạo VM Windows Endpoint

Bước 7.1 — Tạo VM

  1. New → Name: win-agent
  2. Type: Windows, Version: Windows 10/11 (64‑bit)
  3. Memory: 3072–4096 MB
  4. CPU: 2 vCPU
  5. Disk: VDI, Dynamically allocated, size 40–60 GB
  6. Network: chọn giống với wazuh-server (Bridged hoặc NAT+Host‑only)
  7. Mount ISO Windows → cài đặt như bình thường

Bước 7.2 — Cấu hình nhanh sau cài

  • Đặt hostname: win-agent
  • Cập nhật Windows Update
  • Bật Remote Desktop (tuỳ nhu cầu)
  • Cài 7‑Zip/PowerShell 7 (tuỳ thích).
  • Kiểm tra IP: ipconfig

8) Cài Sysmon và cấu hình log nâng cao (Windows)

Sysmon ghi nhận sự kiện chi tiết (tạo process, kết nối mạng, thay đổi registry …) vào Windows Event Log → Wazuh Agent sẽ thu thập về trung tâm.

Bước 8.1 — Cài Sysmon

  1. Tải file Sysmon (bản x64) và giải nén, ví dụ C:\Tools\Sysmon\
  2. Tải file cấu hình mẫu phổ biến sysmonconfig-export.xml (đã lọc nhiễu tốt) và để cùng thư mục.

Bước 8.2 — Cài với cấu hình

Mở PowerShell (Run as Administrator):

Set-Location C:\Tools\Sysmon
.\Sysmon64.exe -i .\sysmonconfig-export.xml -accepteula

Xác minh log đã xuất hiện: Event Viewer → Applications and Services Logs → Microsoft → Windows → Sysmon → Operational.

Mẹo: Có thể cập nhật cấu hình Sysmon khi cần:
.\Sysmon64.exe -c .\sysmonconfig-export.xml

9) Cài Wazuh Agent trên Windows & kết nối về Server

Cách 1 — Dùng lệnh im lặng (nhanh gọn)

  1. Tải file MSI phù hợp, ví dụ: wazuh-agent-4.14.0-1.msi
  2. Mở PowerShell (Admin) trong thư mục chứa file và chạy:
.\wazuh-agent-4.14.0-1.msi /q `
  WAZUH_MANAGER="<IP_WAZUH_SERVER>" `
  WAZUH_AGENT_NAME="win-agent"

Start-Service wazuhsvc

Cách 2 — Làm từ Wazuh Dashboard (khuyên dùng cho người mới)

  • Vào Dashboard → Agents → Deploy new agent → chọn Windows
  • Dashboard sẽ tạo sẵn lệnh cài (kèm tham số manager, agent name, group, secret…) → copy & chạy trên Windows
  • Đợi vài chục giây, agent sẽ xuất hiện trạng thái Active trong Dashboard.

Nếu ghi danh (enroll) lỗi

  • Kiểm tra firewall trên wazuh-server đã mở 1515/TCP và/hoặc 55000/TCP
  • Kiểm tra agent có ping/kết nối TCP tới server không: Test-NetConnection <IP> -Port 1514
  • Xem log trên Windows: C:\Program Files (x86)\ossec-agent\ossec.log

10) Kiểm thử SOC mini: 3 bài test an toàn

Mục tiêu: thấy log/cảnh báo chảy về Wazuh và biết cách truy vấn.

Test A — Tạo process lạ (do Sysmon ghi nhận)

Trên Windows (PowerShell):

# Tạo một script .bat đơn giản rồi chạy
"start calc" | Out-File $env:TEMP\demo.bat -Encoding ascii
Start-Process $env:TEMP\demo.bat

Kỳ vọng: Sysmon Event ID 1 (Process Create) xuất hiện, Wazuh nhận log và hiển thị trong Dashboard.

Test B — Sự kiện đăng nhập thất bại (Windows Security 4625)

  • Khoá màn hình và nhập sai mật khẩu 1–2 lần (an toàn).
  • Mở Dashboard → Security events → lọc theo agent.name: "win-agent" và từ khoá 4625.

Test C — Tạo file đáng ngờ (vô hại)

# Tạo một file .ps1 ngắn và thực thi (chỉ in chuỗi)
"Write-Output 'hello world'" | Out-File $env:TEMP\hello.ps1 -Encoding ascii
powershell.exe -ExecutionPolicy Bypass -File $env:TEMP\hello.ps1

Kỳ vọng: Có sự kiện Process Create và CommandLine chứa ExecutionPolicy Bypass (mẫu hay dùng trong tấn công), dễ thấy trong log để minh hoạ hành vi.

Lưu ý đạo đức: Các bài test trên vô hại và chỉ để học tập. Không chạy mã độc. Không thử nghiệm trên máy không thuộc quyền sở hữu của bạn.

11) Bảo mật & thực hành tốt

  • Cô lập lab: Dùng NAT+Host‑only nếu không cần truy cập từ LAN ngoài.
  • Snapshot sau mỗi mốc: Sau khi cài Ubuntu, sau khi cài Wazuh, sau khi thêm agent.
  • Đổi mật khẩu admin Dashboard ngay lần đầu.
  • UFW: Chỉ mở cổng cần thiết (443, 1514, 1515, 55000).
  • DNS nội bộ: Tạo bản ghi wazuh.lab trỏ tới VM để dễ nhớ.
  • Sao lưu: Giữ file wazuh-install-files.tar (chứa mật khẩu) ở nơi an toàn.
  • Cập nhật có kiểm soát: Với lab, tránh update lớn đột ngột.

12) Khắc phục sự cố thường gặp

1) Dashboard không truy cập được

  • sudo systemctl status wazuh-dashboard (Ubuntu)
  • Kiểm tra cổng 443 lắng nghe: ss -tulpn | grep :443
  • Trình duyệt báo chứng chỉ: Chọn Advanced → Continue (lab); hoặc triển khai Let's Encrypt nếu cần truy cập từ ngoài.

2) Agent Windows không "Active"

  • Kiểm tra port 1514/TCP từ Windows: Test-NetConnection <IP_WAZUH> -Port 1514
  • Xem log: C:\Program Files (x86)\ossec-agent\ossec.log
  • Kiểm tra đồng hồ hệ thống (NTP) – lệch giờ lớn có thể gây rắc rối.

3) Enrollment thất bại

  • Mở 1515/TCP (auto-enrollment) hoặc 55000/TCP (qua API).
  • Thử cách triển khai từ Dashboard → Deploy new agent (điền đúng IP).

4) Chậm/treo

  • Giảm Retention trên Dashboard, tăng RAM/CPU cho VM Wazuh.
  • Dừng bớt dịch vụ Windows nền.

13) Mở rộng lab

  • Thêm Linux agent (Ubuntu/Debian) và macOS agent.
  • Tích hợp Windows Defender log, PowerShell logging, Audit Policy nâng cao.
  • Xây dashboard tuỳ biến theo MITRE ATT&CK.
  • Thêm công cụ mạng như Suricata/Zeek (IDS) để phong phú nguồn log.

14) Phụ lục A — Bảng cổng dịch vụ Wazuh (mặc định, lab)

Thành phần Cổng Giao thức Mục đích
Wazuh Server 1514 TCP Agent connection service
Wazuh Server 1515 TCP Agent enrollment
Wazuh Server 1516 TCP Cluster daemon (không dùng trong lab đơn)
Wazuh Server 55000 TCP Wazuh RESTful API
Wazuh Indexer 9200 TCP API/ingest (nội bộ)
Wazuh Dashboard 443 TCP Giao diện web
Ghi chú: Bạn không cần mở 9200 ra ngoài. Với lab, chỉ cần 443/1514/1515/55000.

15) Phụ lục B — Cheat sheet lệnh

Trên Ubuntu (Wazuh server):

# Cập nhật hệ thống
sudo apt update && sudo apt -y upgrade

# Cài Wazuh all-in-one
curl -sO https://packages.wazuh.com/4.14/wazuh-install.sh
sudo bash ./wazuh-install.sh -a

# In mật khẩu các thành phần
sudo tar -O -xvf wazuh-install-files.tar \
  wazuh-install-files/wazuh-passwords.txt

# Mở cổng UFW
sudo ufw allow 443/tcp
sudo ufw allow 1514/tcp
sudo ufw allow 1515/tcp
sudo ufw allow 55000/tcp
sudo ufw reload

# Kiểm tra dịch vụ
sudo systemctl status wazuh-manager wazuh-dashboard wazuh-indexer
sudo ss -tulpn | egrep ':443|:1514|:1515|:55000|:9200'

Trên Windows (endpoint):

# Cài Sysmon với cấu hình lọc sẵn
Set-Location C:\Tools\Sysmon
.\Sysmon64.exe -i .\sysmonconfig-export.xml -accepteula

# Cài Wazuh Agent (im lặng)
Set-Location C:\Users\Public\Downloads
.\wazuh-agent-4.14.0-1.msi /q WAZUH_MANAGER="<IP_WAZUH_SERVER>" WAZUH_AGENT_NAME="win-agent"
Start-Service wazuhsvc

# Kiểm tra kết nối port 1514
Test-NetConnection <IP_WAZUH_SERVER> -Port 1514

🎉 Kết luận

Vậy là bạn đã dựng xong SOC mini: một Wazuh all‑in‑one giám sát một Windows endpoint với Sysmon. Bài lab này đủ để bạn học cách thu thập log, viết truy vấn, đọc alert, và minh hoạ hành vi tấn công an toàn. Hãy snapshot lại trạng thái "đẹp" và thử thêm agent khác để nâng cấp năng lực SOC của bạn!
