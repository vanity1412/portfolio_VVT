🛡️ Khóa học 1: Securing Hosts, Network, and Edge in AWS

1.1 Bảo mật Host (Host Security)

Khái niệm: Bảo vệ các máy chủ (workloads) như EC2 và container khỏi các mối đe dọa.

Cấu hình Bảo mật Host:

🔧

Cứng hóa AMI (Hardening)

Tắt các dịch vụ không cần thiết, cập nhật bản vá. Sử dụng Patch Manager để tự động vá lỗi.

🔍

Amazon Inspector

Quét lỗ hổng tự động cho EC2 và container. Phát hiện các CVE và cấu hình không an toàn.

🔒

Security Groups

Tường lửa stateful ở cấp độ EC2 instance. Chỉ có quy tắc Allow, không có Deny.

⚖️

Auto Scaling & ELB

Sử dụng Auto Scaling Groups và Elastic Load Balancing để xử lý sự cố và duy trì tính sẵn sàng.

💡 Security Groups - Điểm quan trọng:

  • Stateful: Tự động cho phép traffic trả về (return traffic)
  • Chỉ Allow: Không có quy tắc Deny
  • Gắn vào ENI: Elastic Network Interface của EC2
  • Default: Deny all inbound, Allow all outbound

1.2 Bảo mật Mạng (Network Security)

Nền tảng: VPC (Virtual Private Cloud) là mạng riêng ảo, bao gồm Subnets, Route Tables, và các thành phần bảo mật.

Phân đoạn Mạng (Network Segmentation):

Component Level Type Rules
Security Groups Instance (ENI) Stateful Allow only
NACLs Subnet Stateless Allow & Deny

VPC Endpoints - Truy cập Riêng tư:

Gateway Endpoint

Đặc điểm:

  • ✅ Miễn phí
  • Chỉ hỗ trợ S3 và DynamoDB
  • Sử dụng Route Tables
  • Không cần IP address
Interface Endpoint

Đặc điểm:

  • 💰 Có phí
  • Sử dụng AWS PrivateLink
  • Hỗ trợ nhiều dịch vụ AWS
  • Có ENI với private IP

Giám sát & Lọc Lưu lượng:

📊
VPC Flow Logs

Ghi lại metadata của lưu lượng IP (ai, cái gì, khi nào) đi vào/ra VPC. Lưu trữ vào CloudWatch Logs hoặc S3.

🔍
Traffic Mirroring

Sao chép lưu lượng mạng để phân tích sâu. Hữu ích cho threat detection và troubleshooting.

🛡️
AWS Network Firewall

Dịch vụ tường lửa mạng stateful, quản lý tập trung. Hỗ trợ IDS/IPS capabilities.

⚙️
Firewall Manager

Quản lý tập trung các quy tắc (WAF, Network Firewall, Security Group) trên nhiều tài khoản AWS.

Kết nối Hybrid (On-premises & Cloud):

Hai phương pháp kết nối chính:

  1. AWS Site-to-Site VPN:
    • Kết nối mã hóa qua Internet
    • Nhanh chóng triển khai
    • Chi phí thấp
    • Băng thông phụ thuộc vào Internet
  2. AWS Direct Connect:
    • Đường truyền vật lý riêng (dedicated)
    • Băng thông cao, độ trễ thấp
    • Không mã hóa mặc định (cần VPN hoặc MACsec)
    • Chi phí cao hơn, thời gian triển khai lâu hơn

Khắc phục Sự cố Mạng:

  • Reachability Analyzer: Phân tích đường đi, kiểm tra xem hai tài nguyên có thể kết nối hay không
  • Network Access Analyzer: Phân tích cấu hình để tìm các truy cập mạng không mong muốn

1.3 Bảo mật Edge (Edge Security)

Khái niệm: Bảo vệ các dịch vụ ở rìa mạng (gần người dùng cuối) khỏi các cuộc tấn công.

🌐

Amazon CloudFront

Dịch vụ CDN (Content Delivery Network), giúp giảm tải và bảo vệ lớp ứng dụng. Cache content gần người dùng.

🛡️

AWS WAF

Web Application Firewall (Lớp 7), lọc các cuộc tấn công phổ biến (SQL Injection, XSS). Gắn vào CloudFront, ALB, API Gateway.

🔰

AWS Shield

Standard: Miễn phí, bảo vệ cơ bản Layer 3/4
Advanced: $3000/tháng, bảo vệ nâng cao + 24/7 DDoS Response Team

🌍

Route 53

Dịch vụ DNS có khả năng chống DDoS. Hỗ trợ health checks và routing policies.

🔐 Khóa học 2: Protecting Data, Logging, and Monitoring in AWS

2.1 Bảo vệ Dữ liệu & Mã hóa (Data Protection & Encryption)

Phân loại Dữ liệu (Data Classification):

Amazon Macie - Tự động phát hiện dữ liệu nhạy cảm:

  • Sử dụng Machine Learning để phát hiện PII (Personally Identifiable Information)
  • Quét S3 buckets tự động
  • Tạo findings khi phát hiện dữ liệu nhạy cảm
  • Hỗ trợ compliance (GDPR, HIPAA, PCI DSS)

Mã hóa Dữ liệu:

Loại Mã hóa Mô tả Use Cases
Encryption at Rest Bảo vệ dữ liệu khi lưu trữ trên đĩa S3, EBS, RDS, DynamoDB
Encryption in Transit Bảo vệ dữ liệu khi di chuyển trên mạng TLS/HTTPS, VPN, Direct Connect

AWS KMS (Key Management Service):

Envelope Encryption (Mã hóa Bao bọc):

  1. Bước 1: KMS tạo một "Data Key" (Khóa dữ liệu)
  2. Bước 2: Dữ liệu được mã hóa bằng "Data Key"
  3. Bước 3: "Data Key" được mã hóa bằng khóa KMS (CMK)
  4. Bước 4: Lưu trữ cả dữ liệu đã mã hóa và Data Key đã mã hóa

Lợi ích: Giảm số lượng API calls đến KMS, tăng hiệu suất, giảm chi phí.

# Example: Envelope Encryption với AWS SDK
import boto3

kms = boto3.client('kms')

# 1. Generate Data Key
response = kms.generate_data_key(
    KeyId='arn:aws:kms:region:account:key/key-id',
    KeySpec='AES_256'
)

plaintext_key = response['Plaintext']
encrypted_key = response['CiphertextBlob']

# 2. Encrypt data với plaintext_key
# 3. Store encrypted_data + encrypted_key

AWS CloudHSM:

Khi nào dùng CloudHSM thay vì KMS?

  • Cần tuân thủ FIPS 140-2 Level 3
  • Cần kiểm soát hoàn toàn HSM (Hardware Security Module)
  • Cần single-tenant hardware
  • Cần hỗ trợ các thuật toán mã hóa đặc biệt

Lưu ý: CloudHSM đắt hơn KMS và phức tạp hơn để quản lý.

2.2 Ghi log, Giám sát & Cảnh báo (Logging, Monitoring & Alerting)

Phân biệt Logging vs Monitoring:

Logging (Ghi log)

Ghi lại các sự kiện đã xảy ra (historical data)

Ví dụ:

  • CloudTrail: API calls
  • VPC Flow Logs: Network traffic
  • Application Logs
Monitoring (Giám sát)

Theo dõi hệ thống theo thời gian thực (real-time)

Ví dụ:

  • CloudWatch Metrics: CPU, Memory
  • CloudWatch Alarms: Thresholds
  • Real-time dashboards

Dịch vụ Ghi log (Logging Services):

📝
AWS CloudTrail

Ghi lại MỌI cuộc gọi API trong tài khoản (Ai đã làm gì? Khi nào? Từ đâu?). Đây là dịch vụ then chốt cho kiểm toán và điều tra.

📊
CloudWatch Logs

Nơi tập trung lưu trữ log từ các ứng dụng, EC2 (qua CloudWatch Agent), Lambda, VPC Flow Logs.

🌐
VPC Flow Logs

Ghi log lưu lượng IP. Hữu ích cho phát hiện port scanning, unusual traffic patterns.

Dịch vụ Giám sát & An ninh:

📈

AWS CloudWatch

Dịch vụ giám sát trung tâm. Thu thập Metrics và Logs. Tạo Alarms khi chỉ số vượt ngưỡng.

🔍

Amazon GuardDuty

Threat Detection service. Tự động phân tích CloudTrail, VPC Flow Logs để tìm hoạt động độc hại (crypto mining, malicious IPs).

🛡️

AWS Security Hub

Dashboard trung tâm, tổng hợp findings từ GuardDuty, Inspector, Config, Macie. Cung cấp security score.

⚙️

AWS Config

Theo dõi và ghi lại mọi thay đổi cấu hình tài nguyên. Kiểm tra compliance với Config Rules.

Phân tích & Phản hồi Tự động:

Workflow tự động với EventBridge:

  1. GuardDuty phát hiện sự cố (VD: EC2 instance bị compromise)
  2. GuardDuty gửi Finding đến EventBridge
  3. EventBridge match event với rule
  4. EventBridge kích hoạt Lambda function
  5. Lambda tự động:
    • Cách ly EC2 instance (modify Security Group)
    • Tạo snapshot cho forensics
    • Gửi notification đến SNS
    • Tạo ticket trong ServiceNow/Jira
# Example: Lambda function tự động cách ly EC2
import boto3

ec2 = boto3.client('ec2')
sns = boto3.client('sns')

def lambda_handler(event, context):
    # Parse GuardDuty finding
    instance_id = event['detail']['resource']['instanceDetails']['instanceId']
    
    # 1. Create snapshot
    volumes = ec2.describe_volumes(
        Filters=[{'Name': 'attachment.instance-id', 'Values': [instance_id]}]
    )
    for volume in volumes['Volumes']:
        ec2.create_snapshot(VolumeId=volume['VolumeId'])
    
    # 2. Isolate instance (remove all Security Groups, add isolation SG)
    ec2.modify_instance_attribute(
        InstanceId=instance_id,
        Groups=['sg-isolation']  # Security Group chặn all traffic
    )
    
    # 3. Send notification
    sns.publish(
        TopicArn='arn:aws:sns:region:account:security-alerts',
        Subject='EC2 Instance Isolated',
        Message=f'Instance {instance_id} has been isolated due to security finding'
    )
    
    return {'statusCode': 200}

Phân tích Log:

  • Amazon Athena: Truy vấn dữ liệu (log) lưu trên S3 bằng SQL. Serverless, pay-per-query.
  • Amazon OpenSearch: Tìm kiếm, phân tích và trực quan hóa log real-time. Thay thế cho ELK stack.

Kiểm toán (Auditing):

AWS Audit Manager:

  • Tự động thu thập bằng chứng (evidence) để chứng minh tuân thủ
  • Hỗ trợ frameworks: PCI DSS, HIPAA, GDPR, SOC 2, ISO 27001
  • Tạo audit reports tự động
  • Continuous compliance monitoring

💼 Khóa học 3: Managing Incident Response, IAM, and AWS Service Security

3.1 Phản hồi Sự cố (Incident Response)

Shared Responsibility Model:

AWS Responsibility

Security OF the Cloud:

  • Physical infrastructure
  • Hardware, software, networking
  • Managed services (RDS, Lambda, S3)
  • Global infrastructure
Customer Responsibility

Security IN the Cloud:

  • Data encryption
  • IAM policies
  • OS patching (EC2)
  • Application security
  • Network configuration

AWS Artifact - Compliance Reports:

Tải về báo cáo tuân thủ của AWS:

  • SOC 1, SOC 2, SOC 3 reports
  • ISO 27001, ISO 27017, ISO 27018
  • PCI DSS Attestation of Compliance (AOC)
  • HIPAA Business Associate Addendum (BAA)

Truy cập: AWS Console → AWS Artifact → Reports

Incident Response Metrics:

Metric Mô tả Mục tiêu
MTTD Mean Time to Detect - Thời gian trung bình để phát hiện < 15 phút
MTTR Mean Time to Respond - Thời gian trung bình để phản ứng < 1 giờ

3.2 Bảo mật IAM (IAM Security)

IAM Fundamentals:

IAM quản lý 2 khía cạnh:

  1. Authentication (Xác thực): Bạn là ai?
    • IAM Users (username/password)
    • IAM Roles (temporary credentials)
    • Federated users (SSO)
  2. Authorization (Ủy quyền): Bạn được làm gì?
    • IAM Policies (JSON documents)
    • Resource-based policies
    • Service Control Policies (SCPs)

AWS Organizations & SCPs:

🏢
AWS Organizations

Quản lý tập trung nhiều tài khoản AWS. Consolidated billing, centralized logging.

📁
OUs (Organizational Units)

Nhóm các tài khoản theo môi trường (Dev, Test, Prod) hoặc theo team.

🔒
SCPs (Service Control Policies)

"Vệ sĩ" ở cấp độ tổ chức. Giới hạn quyền tối đa mà các tài khoản thành viên có thể có.

// Example SCP: Cấm sử dụng regions ngoài us-east-1 và eu-west-1
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Deny",
      "Action": "*",
      "Resource": "*",
      "Condition": {
        "StringNotEquals": {
          "aws:RequestedRegion": [
            "us-east-1",
            "eu-west-1"
          ]
        }
      }
    }
  ]
}

Policy Types & Evaluation:

Policy Type Attached To Example
Identity-based User, Group, Role "User A được quyền Đọc S3"
Resource-based Resource (S3, KMS, Lambda) "Bucket này cho phép Account 123 Ghi"

⚠️ Quy tắc đánh giá Policy:

  • Explicit Deny (Từ chối rõ ràng) luôn luôn thắng mọi Allow
  • Nếu không có Allow → Default Deny
  • Nếu có Allow và không có Deny → Allow
  • Nếu có Allow và có Explicit Deny → Deny

AWS Identity Center (SSO):

Single Sign-On Workflow:

  1. User đăng nhập vào Identity Center (một lần)
  2. Identity Center xác thực với Identity Provider (Active Directory, Okta, Google Workspace)
  3. User chọn AWS account và role muốn truy cập
  4. Identity Center gọi STS (Security Token Service) để lấy temporary credentials
  5. User được truy cập vào AWS account với permissions của role đã chọn

Lợi ích: Không cần tạo IAM users, centralized access management, MFA enforcement.

Secrets Management:

AWS Secrets Manager

Features:

  • ✅ Automatic rotation
  • ✅ Integration với RDS, Redshift
  • ✅ Fine-grained access control
  • 💰 Có phí ($0.40/secret/month)
Parameter Store

Features:

  • ✅ Free tier available
  • ✅ Hierarchical storage
  • ⚠️ Manual rotation
  • ⚠️ Limited to 10,000 parameters

IAM Analysis Tools:

🔍
IAM Access Analyzer

Phân tích policies để tìm tài nguyên đang được chia sẻ với external entities. Phát hiện unintended access.

🧪
IAM Policy Simulator

Test policies trước khi apply. Kiểm tra xem một policy có cho phép/từ chối một action cụ thể hay không.

3.3 Bảo mật Dịch vụ AWS (Securing AWS Services)

Bảo mật S3 (S3 Security):

🔒

Object Lock

Ngăn chặn việc sửa đổi hoặc xóa đối tượng. WORM (Write Once Read Many) model. Compliance mode vs Governance mode.

📚

Versioning

Giữ lại các phiên bản cũ của đối tượng khi bị ghi đè hoặc xóa. Bảo vệ khỏi accidental deletion.

🔐

MFA Delete

Yêu cầu MFA khi muốn xóa vĩnh viễn một object version hoặc suspend versioning.

Bảo mật Database (RDS Security):

RDS Proxy - IAM Authentication:

Problem: Hardcoding database passwords trong application code là không an toàn.

Solution: RDS Proxy cho phép sử dụng IAM authentication thay vì passwords.

Benefits:

  • ✅ Không cần lưu passwords trong code
  • ✅ Centralized credential management
  • ✅ Connection pooling (giảm database connections)
  • ✅ Automatic failover (< 30 seconds)
  • ✅ Enhanced security với IAM policies
# Example: Connect to RDS via RDS Proxy với IAM Auth
import boto3
import pymysql

# Generate auth token
rds = boto3.client('rds')
token = rds.generate_db_auth_token(
    DBHostname='proxy-endpoint.proxy-xxx.region.rds.amazonaws.com',
    Port=3306,
    DBUsername='db_user',
    Region='us-east-1'
)

# Connect using token as password
connection = pymysql.connect(
    host='proxy-endpoint.proxy-xxx.region.rds.amazonaws.com',
    user='db_user',
    password=token,
    database='mydb',
    ssl={'ssl': True}
)

🎓 Tổng kết & Key Takeaways

💡 Những điểm quan trọng nhất:

  • ✅ Defense in Depth: Sử dụng multiple layers of security (Network, Host, Application, Data)
  • ✅ Least Privilege: Grant minimum permissions cần thiết cho users và services
  • ✅ Encryption Everywhere: Encrypt data at rest và in transit
  • ✅ Logging & Monitoring: Enable CloudTrail, VPC Flow Logs, CloudWatch cho all critical resources
  • ✅ Automation: Automate security responses với EventBridge + Lambda
  • ✅ IAM Best Practices: Use roles instead of users, enable MFA, rotate credentials
  • ✅ Shared Responsibility: Hiểu rõ trách nhiệm của AWS vs Customer
  • ✅ Incident Response: Have playbooks và automated responses sẵn sàng
