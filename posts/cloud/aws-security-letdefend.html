1️⃣ Introduction to AWS

AWS là gì? / What is AWS?

VN: AWS (Amazon Web Services) là nền tảng điện toán đám mây toàn diện và được sử dụng rộng rãi nhất, cung cấp hơn 200 dịch vụ từ các trung tâm dữ liệu trên toàn cầu.

EN: AWS is a comprehensive and widely-used cloud computing platform provided by Amazon, offering over 200 services from data centers globally.

Các dịch vụ chính / Key Services:

  • Compute: EC2 (Virtual Servers), Lambda (Serverless)
  • Storage: S3 (Object Storage), EBS (Block Storage)
  • Database: RDS (Relational), DynamoDB (NoSQL)

2️⃣ AWS CloudWatch - Giám sát và Cảnh báo

CloudWatch là gì? / What is CloudWatch?

VN: AWS CloudWatch là dịch vụ giám sát và quan sát cho các tài nguyên AWS và ứng dụng. Nó thu thập và theo dõi metrics, logs, và events.

EN: AWS CloudWatch is a monitoring and observability service for AWS resources and applications. It collects and tracks metrics, logs, and events.

Các thành phần chính / Key Components:

1. Metrics (Chỉ số)

  • Dữ liệu time-series về hiệu suất tài nguyên (CPU, Memory, Network)
  • AWS cung cấp metrics mặc định
  • Có thể tạo Custom Metrics bằng CloudWatch API
  • Thu thập mỗi phút (hoặc chi tiết hơn với Detailed Monitoring)

2. Alarms (Cảnh báo)

  • Giám sát metrics và kích hoạt hành động tự động
  • Thiết lập ngưỡng (threshold): VD: CPU > 90% trong 5 phút
  • Actions: Gửi SNS, Stop/Terminate EC2, Invoke Lambda, Auto Scaling

3. Dashboards (Bảng điều khiển)

  • Giao diện trực quan tùy chỉnh
  • Thêm Widgets (metrics, alarms, text/Markdown)
  • Tập trung giám sát nhiều tài nguyên

4. Logs (Nhật ký)

  • Log Groups: Nhóm logic (VD: application logs)
  • Log Streams: Nguồn riêng lẻ (VD: EC2 instance, Lambda)
  • Log Retention Policies: Định nghĩa thời gian lưu trữ
  • CloudWatch Logs Insights: Query và phân tích logs

CloudWatch Logs Insights - Query Examples:

-- Tìm failed login attempts
fields @timestamp, @message 
| filter @message like 'Failed login attempt' 
| sort @timestamp desc 
| limit 10

-- Phát hiện DDoS (nhiều requests từ 1 IP)
fields @timestamp, @ipAddress 
| stats count(*) as requestCount by @ipAddress 
| filter requestCount > 100 
| sort requestCount desc 
| limit 10

-- Tìm API requests chậm
fields @timestamp, @requestDuration 
| filter @requestType = 'API' and @requestDuration > 500 
| sort @timestamp desc 
| limit 20

Use Case: Phát hiện Brute-Force SSH Attack

Scenario thực tế:

  1. Configuration:
    • Custom Metric: Track failed SSH login attempts
    • Alarm: Trigger nếu > 3 failed attempts trong 1 phút
    • SNS Notification: Gửi alert đến SOC team
  2. Response:
    • SOC team nhận alert
    • Dùng Logs Insights query SSH logs, xác định attacker IP
    • Isolate EC2 instance (modify Security Group)
    • Block attacker IP (Network ACLs)
  3. Post-Incident:
    • Enforce strong password policies/MFA
    • Automate IP blocking với Lambda
    • Security training

Best Practices cho CloudWatch:

  • ✅ Sử dụng IAM để kiểm soát truy cập (Principle of Least Privilege)
  • ✅ Enable MFA cho administrative users
  • ✅ Centralize logging với CloudWatch Logs
  • ✅ Enable Log Encryption (AWS KMS)
  • ✅ Set Retention Policies phù hợp
  • ✅ Monitor và alarm trên metrics quan trọng
  • ✅ Tạo Custom Dashboards cho KPIs
  • ✅ Optimize costs: Review usage, xóa alarms không cần thiết
  • ✅ Automate actions với CloudWatch Events/EventBridge

3️⃣ AWS WAF - Web Application Firewall

WAF là gì? / What is WAF?

VN: AWS WAF là dịch vụ firewall bảo vệ ứng dụng web khỏi các cuộc tấn công phổ biến như SQL Injection, XSS, và DDoS.

EN: AWS WAF is a web application firewall that helps protect web applications from common web exploits like SQL injection, XSS, and DDoS attacks.

Khái niệm cơ bản / Basic Concepts:

1. Web ACLs (Access Control Lists)

  • Định nghĩa rules và conditions để Allow, Block, hoặc Count traffic
  • Gắn vào CloudFront, ALB, API Gateway, hoặc AppSync
  • Components: Rules + Rule Actions

2. Managed Rule Groups

  • Bộ rules được AWS security experts tạo và maintain
  • VD: "SQL Injection", "XSS", "Known Bad Inputs"
  • Dễ dàng áp dụng, không cần viết rules từ đầu

3. Custom Rules

  • Tùy chỉnh theo vulnerabilities cụ thể của ứng dụng
  • Dựa trên IP addresses, headers, query strings, body, etc.

Advanced Configurations:

Rate-Based Rules (Chống Brute-Force & DDoS)

Rule Type: Rate-based
Rate Limit: 100 requests per 5 minutes
Action: Block
Response: 
  Status Code: 500
  Custom Header: "You: Blocked"

Regex Pattern Matching (Chống SQL Injection)

Pattern: .*(SELECT|INSERT|UPDATE|DELETE|UNION|DROP).*
Match Type: Contains SQL Injection Attack
Action: Block
Response Code: 502
Header: "Blocked: Possible SQL Injection"

IP Set Matching

  • Tạo IP Sets chứa danh sách IP addresses/ranges
  • Allow traffic từ trusted IPs (whitelist)
  • Block traffic từ malicious IPs (blacklist)

Blocking SQL Injection - Step by Step:

  1. Access AWS WAF Console
  2. Create Web ACL → Attach to resources (CloudFront/ALB)
  3. Create Rule:
    • Rule Type: Regular rule
    • Match Type: Contains SQL Injection Attack
    • Inspect: Query string, Body, Headers
  4. Configure Action:
    • Action: Block
    • Custom Response: 502 status code
    • Header: "Blocked: Possible SQL Injection"
  5. Review and Activate

Best Practices cho WAF:

  • ✅ Rule Optimization: Regularly review rules, remove outdated ones
  • ✅ Use Whitelists: Cho known partners/internal services
  • ✅ Enable Logging: Store logs in S3, analyze với Athena/QuickSight
  • ✅ Set Up CloudWatch Alarms: Monitor blocked requests, rule matches
  • ✅ Test Rule Changes: Dùng staging environment hoặc Web ACL simulator
  • ✅ Stay Informed: Subscribe to AWS security bulletins
  • ✅ Schedule Regular Reviews: Ít nhất quarterly

💡 Tại sao WAF quan trọng? / Why WAF Matters?

  • Protection: Chống common web exploits (OWASP Top 10)
  • DDoS Mitigation: Rate-based rules giới hạn requests
  • Compliance: Đáp ứng yêu cầu bảo mật (PCI DSS, HIPAA)
  • Flexible: Customizable rules cho từng ứng dụng
  • Real-time Monitoring: Insights và metrics qua CloudWatch
  • Cost-effective: Pay-as-you-go pricing

4️⃣ AWS GuardDuty - Threat Detection Service

GuardDuty là gì? / What is GuardDuty?

VN: AWS GuardDuty là dịch vụ phát hiện mối đe dọa được quản lý hoàn toàn, sử dụng Machine Learning và Threat Intelligence để bảo vệ AWS environment.

EN: AWS GuardDuty is a fully managed threat detection service that uses Machine Learning and Threat Intelligence to protect your AWS environment.

Data Sources được phân tích:

  • AWS CloudTrail Logs: API calls và user activities
  • VPC Flow Logs: Network traffic patterns
  • DNS Logs: DNS queries (phát hiện phishing, malware)
  • External Threat Feeds: Global threat intelligence

Các loại threats được phát hiện:

  • 🔴 Unauthorized Access: Unusual logins, suspicious API calls
  • 🔴 Cryptocurrency Mining: Compromised instances mining crypto
  • 🔴 Data Exfiltration: Unusual data transfer patterns
  • 🔴 Malware & Trojan: Communication với C&C servers
  • 🔴 Brute Force Attacks: Repeated login attempts

Severity Levels:

  • Low: Potentially suspicious, nên investigate
  • Medium: More suspicious, cần prompt investigation
  • High: Critical threats, cần immediate attention

Setting Up GuardDuty:

  1. Enable GuardDuty:
    • AWS Console → GuardDuty
    • Click "Enable GuardDuty"
    • Chọn AWS Regions để monitor
  2. Configure Data Sources:
    • GuardDuty tự động access CloudTrail, VPC Flow Logs, DNS Logs
    • Optional: Enable Lambda Protection, EKS Protection
  3. Customize Settings:
    • Adjust Threat Intelligence Feeds
    • Set Finding Thresholds
    • Configure SNS notifications
    • Specify S3 bucket để store findings
  4. Automated Responses:
    • Integrate với Lambda functions
    • Auto-remediation workflows

Interpreting GuardDuty Findings:

Anatomy of a Finding:

  • Title: Summary của threat
  • Description: Chi tiết insights
  • Severity: Low/Medium/High
  • Resource Affected: EC2, S3, RDS, etc.
  • Additional Context: IP addresses, API calls, timestamps
  • Recommended Actions: Guidance cho incident response

Real-World Examples:

Example 1: Suspicious File Activity (EICAR-Test-File)

Threat: GuardDuty detected eicar.com (Severity: MEDIUM)

Analysis: EICAR test file indicates capability to recognize malware patterns

Action:

  • Isolate system
  • Remove file
  • Conduct security audit
  • Review antivirus configurations

Example 2: Unauthorized RDS Access

Threat: IP 1.2.3.4 (known malicious) attempted unsuccessful login to RDS (Severity: MEDIUM)

Action:

  • Investigate RDS access logs
  • Review IAM policies
  • Implement MFA
  • IP whitelisting
  • Update Security Groups

Example 3: Port Scan Detection

Threat: EC2 instance i-99999999 attempted OUTBOUND port scan to 198.51.100.0:797 (Severity: MEDIUM)

Action:

  • Investigate instance for compromise
  • Review outbound traffic rules
  • Enhance Security Groups/NACLs
  • Check for malware/backdoors

Investigation Process:

  1. Incident Context: Gather timeline, affected resources
  2. Forensic Analysis: Deep dive vào logs, network traffic
  3. Collaboration: Work với other teams (DevOps, Security)
  4. Remediation: Isolate instance, revoke access, patch vulnerabilities
  5. Documentation: Document process và lessons learned

Handling False Positives:

  • ✅ Tuning Findings: Adjust detection thresholds
  • ✅ Contextual Analysis: Differentiate genuine threats vs. legitimate use cases
  • ✅ Regular Review: Validate findings, identify recurring false positives
  • ✅ Feedback Loop: Report false positives để refine GuardDuty

5️⃣ AWS CloudTrail - Audit và Compliance

CloudTrail là gì? / What is CloudTrail?

VN: AWS CloudTrail là dịch vụ ghi lại mọi API call được thực hiện trong AWS account, cung cấp audit trail cho compliance và security analysis.

EN: AWS CloudTrail is a service that records every API call made in your AWS account, providing an audit trail for compliance and security analysis.

Tại sao CloudTrail quan trọng?

  • 🔍 Enhanced Security: Phát hiện unusual/unauthorized activities
  • 📋 Compliance & Governance: Detailed record cho audits
  • 🔧 Operational Insights: Troubleshoot và root cause analysis
  • 📊 Resource Tracking: Track changes (who created/modified/deleted)

Key Features:

  • Comprehensive Logging: Captures requester identity, API called, parameters, response
  • Universal Coverage: Records activity across all AWS services
  • Integration: CloudWatch Logs, Lambda, S3
  • Data Encryption: Encrypts log files at rest
  • Multi-Region Support: Deliver logs từ multiple regions vào single S3 bucket

CloudTrail Log Structure:

{
  "eventVersion": "1.08",
  "eventTime": "2025-11-14T10:30:00Z",
  "eventSource": "signin.amazonaws.com",
  "eventName": "ConsoleLogin",
  "userIdentity": {
    "type": "Root",
    "principalId": "123456789012",
    "arn": "arn:aws:iam::123456789012:root"
  },
  "sourceIPAddress": "217.131.99.238",
  "responseElements": {
    "ConsoleLogin": "Success"
  },
  "additionalEventData": {
    "MFAUsed": "No"
  }
}

Interpreting CloudTrail Events:

Key Skills:

  • Deciphering Event Names: CreateBucket, RunInstances, DeleteUser
  • Analyzing User Identity: Identify IAM user/role thực hiện action
  • Utilizing Timestamps: Establish timelines cho incidents
  • Resource-Level Logging: Log data events (S3 object uploads, Lambda invocations)

Example: Root User Login Without MFA

Event: Root User successfully logged in từ IP 217.131.99.238 WITHOUT MFA

Severity: 🔴 HIGH (Root account không nên dùng thường xuyên)

Recommended Actions:

  • Enable MFA cho Root Account ngay lập tức
  • Investigate IP address (legitimate?)
  • Review CloudTrail logs cho actions performed
  • Rotate Root credentials nếu cần
  • Educate team về Root account best practices

Detecting Login Issues với CloudTrail:

Key Fields:

  • eventSource: signin.amazonaws.com
  • eventName: ConsoleLogin
  • responseElements.ConsoleLogin: "Success" hoặc "Failure"
  • additionalEventData.MFAUsed: "Yes" hoặc "No"
  • userIdentity.type: "Root" hoặc "IAMUser"

SQL Query Example (Athena):

SELECT
  eventTime,
  userIdentity.arn AS user,
  awsRegion,
  userAgent,
  sourceIPAddress,
  eventName,
  responseElements.ConsoleLogin AS loginStatus,
  additionalEventData.MFAUsed AS mfaUsed
FROM cloudtrail_logs
WHERE
  eventName = 'ConsoleLogin'
  AND eventTime > '2025-11-01 00:00:00'
ORDER BY eventTime DESC;

Threats có thể phát hiện:

  • 🔴 Brute-force login attempts: Multiple "Failure" từ same IP
  • 🔴 Users logging in without MFA: MFAUsed: "No"
  • 🔴 Logins từ malicious IPs: Cross-reference với threat intelligence
  • 🔴 Unusual API calls: DeleteBucket, TerminateInstances ngoài giờ
  • 🔴 Privilege escalation: AttachUserPolicy, CreateAccessKey

Integration với SIEM (Splunk):

  1. CloudTrail logs được lưu ở S3 (JSON format)
  2. Install Splunk Add-on for AWS (ingestion)
  3. Install Splunk App for AWS (dashboards)
  4. Configure S3 bucket access
  5. Create dashboards và alerts trong Splunk

Best Practices:

  • ✅ Enable CloudTrail trong tất cả regions
  • ✅ Enable Log File Validation (detect tampering)
  • ✅ Store logs trong dedicated S3 bucket với encryption
  • ✅ Enable Data Events cho critical S3 buckets
  • ✅ Integrate với CloudWatch Logs cho real-time monitoring
  • ✅ Set up CloudWatch Alarms cho suspicious activities
  • ✅ Regularly review logs (automated analysis với Athena/Lambda)
  • ✅ Implement S3 Lifecycle policies để manage log retention

6️⃣ AWS Incident Management

AWS Incident Manager là gì?

VN: AWS Incident Manager là tính năng của Systems Manager, giúp giảm thời gian giải quyết incidents thông qua automated plans, runbooks, và collaboration tools.

EN: AWS Incident Manager is a feature of Systems Manager that helps decrease incident resolution time through automated plans, runbooks, and collaboration tools.

Key Components:

  • Response Plan: Định nghĩa who responds, automation runbooks, communication tools
  • Incident Detection: CloudWatch alarms và EventBridge events tạo incidents
  • Runbook Automation: Tự động thực hiện remediation actions
  • Engagement Plans: Notify đúng người đúng lúc
  • On-call Schedules: Rotas (up to 30 people)
  • Collaboration: AWS Chatbot (Slack, MS Teams, Chime)
  • Post-Incident Analysis: Review và continuous improvement

Real-World Example: Brute-Force Attack Response

Situation:

CloudWatch Alarm phát hiện abnormal failed login attempts

Setup:

  • CloudWatch Alarm monitors access logs
  • Response Plan:
    1. Notify security team via SMS
    2. Execute Automation runbook to isolate resources

Response Workflow:

  1. Alarm triggers → Incident Manager activates Response Plan
  2. Security team notified via SMS
  3. AWS Chatbot connects team to chat channel
  4. Runbook executes automatically: Isolate compromised resources
  5. OpsCenter provides centralized tracking

Key Learnings:

  • ✅ Automated response reduces manual intervention
  • ✅ Chatbot facilitates real-time collaboration
  • ✅ Predefined runbooks ensure consistency
  • ✅ OpsCenter provides centralized visibility

7️⃣ Amazon VPC - Virtual Private Cloud

VPC là gì?

VN: Amazon VPC cho phép bạn tạo một mạng ảo riêng biệt trong AWS cloud, với full control về IP addressing, subnets, route tables, và gateways.

EN: Amazon VPC allows you to create a logically isolated virtual network in AWS cloud, with complete control over IP addressing, subnets, route tables, and gateways.

Key Components:

1. Subnets

  • Public Subnets: Có route đến Internet Gateway (cho web servers)
  • Private Subnets: Không có direct internet access (cho databases)
  • Mỗi subnet nằm trong 1 Availability Zone

2. Route Tables

  • Control traffic routing giữa subnets
  • 0.0.0.0/0 route → Internet Gateway (public subnet)
  • Routes evaluated based on most specific match

3. Security Groups (Stateful)

  • Virtual firewalls ở instance level
  • Tracks connection state (allows return traffic automatically)
  • Default: Allow all outbound, block all inbound

4. Network ACLs (Stateless)

  • Network-level security ở subnet level
  • Rules processed in order (lowest to highest number)
  • Specify Allow/Deny cho inbound và outbound

5. NAT Gateway

  • Provide outbound internet connectivity cho private subnets
  • Public NAT: Requires Elastic IP, routes to Internet Gateway
  • Private NAT: Routes through transit/virtual private gateways

6. Internet Gateway

  • Facilitates communication giữa VPC và internet
  • Highly available, horizontally scaled
  • Only 1 Internet Gateway per VPC

VPC Flow Logs:

Gathers details về IP traffic giữa network interfaces trong VPC

2 eni-0d14a8f66abcd1234 172.31.16.21 203.0.113.10 49234 443 6 10 3210 1637044452 1637044512 REJECT OK

Fields: Version, ENI, Src IP, Dst IP, Src Port, Dst Port, Protocol (6=TCP), Packets, Bytes, Start Time, End Time, Action (REJECT), Status

Use Case: Detecting Port Scanning

Pattern: Multiple flows từ same source IP (192.168.1.200) đến various destination ports (22, 80, 443, 3389) với REJECT action

Indication: Potential port scanning attack

Action: Block source IP, investigate compromised systems

8️⃣ AWS S3 Security

S3 Security Best Practices:

1. Restrict Public Access

  • Activate S3 Block Public Access ở account/bucket level
  • Use AWS Organizations SCPs để enforce

2. Enable S3 Protection in GuardDuty

  • Analyze S3 data access events
  • Detect suspicious activities (uncommon locations, deactivation of controls)

3. Use Amazon Macie

  • Machine learning để detect sensitive data
  • Notifies về public/unencrypted/externally shared buckets

4. Encryption

  • SSE-S3: S3-managed keys (simple)
  • SSE-KMS: KMS-managed keys (audit trail, enhanced protection)
  • SSE-C: Customer-provided keys
  • Client-side: Encrypt before sending to AWS

5. Versioning & Object Lock

  • S3 Versioning: Preserves every version
  • MFA Delete: Requires 2FA to delete versions
  • Object Lock: WORM model (write-once-read-many)

6. Logging

  • CloudTrail: API calls (enable Data Events)
  • Server Access Logging: Detailed request records

7. Cross-Region Replication

  • Automatic copying across regions
  • For compliance và disaster recovery

9️⃣ AWS IAM - Identity and Access Management

IAM Components:

1. Users

  • Individual users với unique credentials
  • Programmatic access: Access Key + Secret Key
  • Console access: Username + Password

2. Groups

  • Organize users để manage permissions collectively
  • Users inherit group permissions
  • Users có thể belong to multiple groups

3. Roles

  • Set of permissions mà entities có thể assume
  • Used for temporary access hoặc cross-account access
  • Example: Lambda role để access S3

4. Policies

  • JSON documents defining permissions
  • Effect: "Allow" hoặc "Deny"
  • Action: AWS service API actions
  • Resource: AWS resources (ARNs)

Policy Example:

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}

Best Practices:

  • ✅ Principle of Least Privilege: Grant minimum permissions
  • ✅ Enable MFA: Especially cho Root và admin users
  • ✅ Rotate Access Keys: Regularly (every 90 days)
  • ✅ Use IAM Roles: Instead of access keys khi possible
  • ✅ Avoid Hardcoding Keys: Use Secrets Manager/Parameter Store
  • ✅ Monitor Key Usage: CloudTrail logs
  • ✅ Use Groups: Manage permissions collectively
  • ✅ Regular Audits: Review permissions và policies

🔟 AWS Shield - DDoS Protection

AWS Shield Standard vs Advanced:

Feature Shield Standard Shield Advanced
Cost Free (automatic) $3000/month + fees
Protection Layer Layer 3 & 4 Layer 3, 4, & 7
Detection Static thresholds Personalized (traffic patterns)
WAF Integration Manual Automatic L7 mitigation
Support Standard AWS Support 24/7 Shield Response Team
Cost Protection No Yes (DDoS scaling charges)

DDoS Attack Types:

  • Volume-based: Massive burst of requests (overwhelms bandwidth)
  • Protocol attacks: Malformed requests exploit network protocols
  • Application-layer (L7): Complex requests target app vulnerabilities

  • ✅ Automation: Automate detection và response với CloudWatch, GuardDuty, Incident Manager
  • ✅ Least Privilege: Grant minimum permissions cần thiết
  • ✅ Monitoring: Enable logging và monitoring cho all critical resources
  • ✅ Encryption: Encrypt data at rest và in transit
  • ✅ Regular Audits: Review permissions, policies, và configurations
  • ✅ Incident Response: Have plans và runbooks sẵn sàng

  2. Tham gia AWS Security challenges trên Let's Defend
  3. Học thêm về AWS Security Specialty Certification
  4. Build personal projects áp dụng security best practices
  5. Join AWS Security community và forums
