Tóm tắt Week 2 về Database & Storage trên AWS - RDS, DynamoDB, S3, EBS

AWS Database Storage RDS S3

Chào các bạn! Mình là Vũ Văn Thông, sinh viên năm 3 ngành An toàn thông tin. Hôm nay mình muốn chia sẻ với các bạn những gì mình đã học được về Database & Storage Foundation trên AWS (Week 2). Đây là tổng hợp kiến thức về các dịch vụ database và storage quan trọng trên AWS!

1) Giới thiệu về Database & Storage trên AWS

AWS cung cấp nhiều dịch vụ database và storage khác nhau, mỗi loại phù hợp với một use case cụ thể. Mình sẽ tóm tắt các dịch vụ chính:

Các dịch vụ chính:

  • RDS - Relational Database Service (SQL databases)
  • DynamoDB - NoSQL database service
  • S3 - Simple Storage Service (object storage)
  • EBS - Elastic Block Store (block storage)
  • EFS - Elastic File System (file storage)

2) Amazon RDS (Relational Database Service)

RDS là dịch vụ quản lý database quan hệ (SQL) trên AWS. Nó hỗ trợ nhiều database engines như MySQL, PostgreSQL, Oracle, SQL Server, và MariaDB.

Ưu điểm của RDS:

  • Tự động backup và restore
  • Multi-AZ deployment cho high availability
  • Read replicas để scale read operations
  • Automatic patching và updates
  • Monitoring với CloudWatch

Use cases:

  • Web applications cần SQL database
  • E-commerce platforms
  • CRM systems
  • ERP applications

3) Amazon DynamoDB

DynamoDB là dịch vụ NoSQL database được quản lý hoàn toàn, cung cấp hiệu suất cao và khả năng scale tự động.

Đặc điểm chính:

  • Fully managed NoSQL database
  • Single-digit millisecond latency
  • Automatic scaling
  • Built-in security và backup
  • Global tables cho multi-region

Use cases:

  • Mobile và web applications
  • Gaming applications
  • IoT applications
  • Real-time bidding

4) Amazon S3 (Simple Storage Service)

S3 là dịch vụ object storage với khả năng scale không giới hạn, được thiết kế để lưu trữ và truy xuất bất kỳ lượng dữ liệu nào.

Storage Classes:

  • S3 Standard - Frequently accessed data
  • S3 Intelligent-Tiering - Automatic cost optimization
  • S3 Standard-IA - Infrequently accessed data
  • S3 Glacier - Long-term archive
  • S3 Glacier Deep Archive - Lowest cost archive

Use cases:

  • Backup và archive
  • Data lakes
  • Static website hosting
  • Media storage và distribution

5) Amazon EBS (Elastic Block Store)

EBS cung cấp block storage volumes cho EC2 instances. Nó giống như một hard drive ảo gắn vào EC2 instance.

Volume Types:

  • gp3/gp2 - General Purpose SSD
  • io2/io1 - Provisioned IOPS SSD
  • st1 - Throughput Optimized HDD
  • sc1 - Cold HDD

Use cases:

  • Boot volumes cho EC2
  • Databases
  • Enterprise applications
  • Big data analytics

6) Amazon EFS (Elastic File System)

EFS là dịch vụ file storage có thể scale tự động, được thiết kế cho Linux workloads.

Đặc điểm:

  • Fully managed NFS file system
  • Automatic scaling
  • Multi-AZ availability
  • Concurrent access từ nhiều EC2

7) So sánh các dịch vụ Storage

Khi nào dùng gì?

  • S3 - Object storage, static files, backups
  • EBS - Block storage cho EC2, databases
  • EFS - Shared file storage cho nhiều EC2
  • RDS - SQL databases được quản lý
  • DynamoDB - NoSQL với low latency

8) Best Practices

Security:

  • Enable encryption at rest và in transit
  • Sử dụng IAM policies để control access
  • Enable versioning cho S3
  • Regular backups và snapshots

Cost Optimization:

  • Chọn đúng storage class cho use case
  • Sử dụng lifecycle policies
  • Delete unused volumes và snapshots
  • Monitor usage với CloudWatch

Kết luận

Database và Storage là hai thành phần quan trọng trong bất kỳ ứng dụng nào trên AWS. Việc hiểu rõ các dịch vụ và biết khi nào nên dùng dịch vụ nào sẽ giúp bạn thiết kế hệ thống tốt hơn và tối ưu chi phí.
